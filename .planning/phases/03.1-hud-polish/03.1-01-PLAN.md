pub fn compute_hud_layout(screen_w: f32, screen_h: f32) -> HudLayout {
    let scale_x = screen_w / 640.0;
    let scale_y = screen_h / 480.0;
    let sidebar_w = (100.0 * scale_x).round();   // <-- TOO NARROW: 100px at 640x480

4. Include the three new fields in the HudLayout return struct.

**Part B: Fix draw_hud rendering in app.rs**

1. Remove the hardcoded mana_bar_y calculation (line ~1891). Use `layout.mana_bar_y` instead.
2. Remove the hardcoded mana_bar_h calculation (line ~1892). Use `layout.mana_bar_h` instead.
3. For mana text, render INSIDE the mana bar at `layout.mana_bar_y + 1.0` (keep existing).
4. For population, replace `player_pop_y` calculation with `layout.pop_y`.
5. For the spell panel (HudTab::Spells match arm, lines ~1857-1873), switch from 2-column to 1-column layout:
   ```rust
   HudTab::Spells => {
       for (i, entry) in hud_state.panel_entries.iter().enumerate() {
           let sy = layout.panel_y + i as f32 * layout.line_h;
           let entry_w = layout.sidebar_w - layout.mm_pad * 2.0;
           // Cooldown overlay
           let cooldown = hud_state.spell_cooldowns.iter().find(|c| c.spell_index == i as u8);
           if let Some(cd) = cooldown {
               if cd.cooldown_remaining > 0 {
                   let cd_frac = cd.cooldown_remaining as f32 / cd.cooldown_total.max(1) as f32;
                   hud.draw_rect(layout.mm_pad, sy, entry_w, layout.line_h, [0.0, 0.0, 0.0, 0.5 * cd_frac]);
               }
           }
           hud.draw_text(&entry.label, layout.mm_pad, sy, layout.small_font, entry.color);
       }
   }
   ```
   This removes the `row/col` 2-column indexing and renders each spell on its own full-width line.

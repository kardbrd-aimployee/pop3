  <action>
    Replace the placeholder `pub struct ObjectPool;` in pool.rs with a full implementation.

    IMPORTANT: Per user decision, this is a MODERN Rust-idiomatic pool, NOT a faithful replica of the original binary's two-tier free list. Use a single simple free list through the existing PoolSlot::Free variant. No high-priority/low-priority split needed.

    Implementation:
    1. `ObjectPool` struct with fields:
       - `slots: Box<[PoolSlot; MAX_OBJECTS]>` (heap-allocated, ~400KB)
       - `free_head: Option<u16>` (single free list head)
       - `active_count: u16`

    2. `ObjectPool::new()` — initialize all slots as Free, linking 0->1->2->...->1100->None. free_head = Some(0).

    3. `ObjectPool::create(model_type, subtype, tribe, position) -> Option<ObjectHandle>`:
       - Pop from free_head
       - Build ObjectHeader with passed params (health=0, max_health=0, angle=0, flags=0, velocity=default, object_index=slot_idx)
       - Build GameObjectData variant based on model_type (Person -> PersonData::default(), others -> unit tuple ())
       - Store as PoolSlot::Occupied, increment active_count
       - Return Some(slot_idx)

    4. `ObjectPool::destroy(handle) -> bool`:
       - Verify slot is Occupied, replace with Free { next_free: self.free_head }
       - Set free_head = Some(handle), decrement active_count
       - Clear cell linkage (next_in_cell, prev_in_cell) before destroying — caller is responsible for cell grid removal BEFORE calling destroy
       - Return true on success, false if already free or out of range

    5. `ObjectPool::get(handle) -> Option<&GameObject>` — bounds check + occupied check
    6. `ObjectPool::get_mut(handle) -> Option<&mut GameObject>` — same, mutable
    7. `ObjectPool::active_count() -> u16`

    8. `ObjectPool::persons() -> impl Iterator<Item = (ObjectHandle, &ObjectHeader, &PersonData)>`:
       - Iterate all slots, filter Occupied + Person variant, yield (index, header, person_data)

    9. `ObjectPool::persons_mut() -> impl Iterator<Item = (ObjectHandle, &mut ObjectHeader, &mut PersonData)>`:
       - Same but mutable references

    Also update mod.rs to export MAX_OBJECTS from pool.

    Write tests FIRST (RED), then implement (GREEN), then refactor if needed.
  </action>

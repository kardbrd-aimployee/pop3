    - modify_height_area applies modify_height to a radius around center
    - terrain_cascade takes affected cell range and updates: normals, walkability, water, pathfinding segments

  <action>
    Create src/engine/terrain/ module with 4 files:

    **modify.rs**:
    - `pub fn modify_height(heights: &mut [[u16; 128]; 128], x: usize, y: usize, target: u16, rate: u16) -> bool` -- moves heights[y][x] toward target by at most rate per call. Returns true if height changed. Wraps x,y with `& 127`.
    - `pub fn modify_height_area(heights: &mut [[u16; 128]; 128], cx: usize, cy: usize, radius: usize, target: u16, rate: u16) -> bool` -- applies modify_height to all cells within radius of (cx,cy), toroidal distance. Returns true if any height changed.
    - `pub fn flatten_area(heights: &mut [[u16; 128]; 128], cx: usize, cy: usize, radius: usize) -> bool` -- computes average height in area, then modify_height_area toward that average. Returns true if any changed.
    - After any modification that returns true, caller must invoke terrain_cascade.

    **cascade.rs**:
    - `pub struct CascadeRegion { pub min_x: usize, pub min_y: usize, pub max_x: usize, pub max_y: usize }` -- affected cell range (wrapping handled internally)
    - `pub fn terrain_cascade(heights: &[[u16; 128]; 128], region: &CascadeRegion, normals: &mut [[[f32; 3]; 128]; 128], walkability: &mut [[u8; 128]; 128], water_level: u16, segment_pool: &mut SegmentPool) -> CascadeResult`
    - CascadeResult contains: `normals_updated: bool, walkability_changed: Vec<(usize, usize)>, water_cells_changed: Vec<(usize, usize)>, segments_invalidated: u16, mesh_dirty: bool`
    - Cascade steps in order:
      1. Recalculate normals for cells in region (cross-product of adjacent height differences)
      2. Update walkability flags (steep slopes become unwalkable, water cells unwalkable)
      3. Update water cells (call update_water_cells)
      4. Invalidate pathfinding segments (call invalidate_segments_in_region)
      5. Mark mesh as dirty (return flag for renderer)
    - `pub fn invalidate_segments_in_region(segment_pool: &mut SegmentPool, region: &CascadeRegion) -> u16` -- iterate all active segments (ref_count > 0) in SegmentPool. For each segment, check if its src or dst TileCoord falls within the CascadeRegion (using toroidal wrapping). If so, set ref_count to 0 (invalidate the cached path) and decrement active_count. Returns count of invalidated segments. This forces units passing through modified terrain to re-pathfind.

    **water.rs**:
    - `pub fn update_water_cells(heights: &[[u16; 128]; 128], water_flags: &mut [[u8; 128]; 128], region: &CascadeRegion, water_level: u16) -> Vec<(usize, usize)>` -- for each cell in region, if height < water_level set water flag, else clear it. Returns list of changed cells.
    - `pub fn is_water_cell(height: u16, water_level: u16) -> bool` -- height < water_level
    - Constants: `WATER_WALKABILITY_FLAG: u8 = 0x04` (cell is water)

    **mod.rs**:

    ```rust
    pub use modify::{modify_height, modify_height_area, flatten_area};
    pub use cascade::{terrain_cascade, CascadeRegion, CascadeResult, invalidate_segments_in_region};
    pub use water::{update_water_cells, is_water_cell};
    ```

    Normal calculation for cascade: for cell (x,y), compute normal as cross product of (height[y][x+1]-height[y][x-1], 2.0, 0) x (0, 2.0, height[y+1][x]-height[y-1][x]), normalized. Use wrapping indices (& 127).

    Walkability slope threshold: if abs difference between adjacent cells > 512 (height units), mark as unwalkable steep slope.

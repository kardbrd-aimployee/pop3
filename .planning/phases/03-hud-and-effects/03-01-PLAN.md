```rust
pub struct StringTable {
    strings: Vec<String>,
}

impl StringTable {
    /// Parse string table from raw binary data (LANGUAGE/lang00.dat format).
    /// Format: u32le count, then count x u32le offsets, then null-terminated ASCII strings.
    /// Original has 0x526 (1318) strings for English.
    pub fn from_bytes(data: &[u8]) -> Self { ... }

    /// Get string by index. Returns None if out of bounds.
    pub fn get(&self, index: usize) -> Option<&str> { ... }

    /// Number of strings loaded.
    pub fn len(&self) -> usize { ... }
}
```

Binary format:
- Bytes 0-3: u32le string count
- Bytes 4..(4 + count*4): u32le offset array (each offset is from start of file)
- Remaining bytes: null-terminated ASCII strings at the given offsets

Add `pub mod strings;` to `src/data/mod.rs`.

Handle edge cases: data shorter than 4 bytes, offset pointing past data end, missing null terminator (read to end of data).

```rust
pub struct FontGlyph {
    pub width: u32,
    pub height: u32,
    pub bitmap: Vec<u8>,  // 1 byte per pixel (0=transparent, 255=opaque)
}

pub struct FontData {
    pub glyph_width: u32,
    pub glyph_height: u32,
    glyphs: Vec<FontGlyph>,  // 128 entries (ASCII)
}

impl FontData {
    /// Create FontData from the existing 8x8 bitmap font data.
    /// This reuses the FONT_DATA constant already embedded in hud/mod.rs.
    /// Takes the raw 128*8 byte array (1 byte per row, 8 rows per glyph, 128 glyphs).
    pub fn from_8x8_bitmap(data: &[u8; 1024]) -> Self { ... }

    /// Get glyph for ASCII character. Returns blank glyph for out-of-range.
    pub fn glyph(&self, ch: u8) -> &FontGlyph { ... }

    /// Create a scaled version (each pixel becomes scale x scale block).
    /// scale=1 -> 8x8, scale=2 -> 16x16, scale=3 -> 24x24.
    /// These produce three distinct rendered sizes for small/medium/large text.
    pub fn scaled(&self, scale: u32) -> Self { ... }
}
```

Three distinct font sizes via integer scaling of the 8x8 base:
- **Small** (8x8): scale=1 -- for dense info, tooltips
- **Medium** (16x16): scale=2 -- for standard HUD text, labels
- **Large** (24x24): scale=3 -- for headings, important text

The research notes: "Start with the 8x8 bitmap (already working) at scaled sizes. Load original fonts as a follow-up if text quality is insufficient."

Add `pub mod font;` to `src/data/mod.rs`.

Note: The existing FONT_DATA in hud/mod.rs is the raw 8x8 bitmap. The FontData struct provides a cleaner API for multi-size text. Original .fon file loading can be added later if needed.

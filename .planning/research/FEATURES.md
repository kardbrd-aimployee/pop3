# Feature Research

**Domain:** RTS/God-game reimplementation (Populous: The Beginning)
**Researched:** 2026-03-17
**Confidence:** HIGH

## Context

Pop3 is at ~25% completion (91/366 items DONE or PARTIAL). The engine layer is solid: rendering, pathfinding, movement, sprite/animation, camera, and level loading all work. What remains is the gameplay layer -- the systems that turn a tech demo into a playable game. This research categorizes every remaining system by whether it is required for a playable single-player campaign, a differentiator beyond the original, or something to explicitly defer.

The original game has 25 campaign levels with progressive unlocks. A "playable campaign" means: the player can start level 1, build, fight, cast spells, win or lose, and progress through all 25 levels with AI opponents behaving recognizably.

## Feature Landscape

### Table Stakes (Must Have for Playable Campaign)

These are the systems without which no single level can be completed. Missing any one of these means the game is not playable.

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| **Core Object System** (unified pool, cell grid) | Every gameplay entity depends on this; buildings, spells, effects all need proper object lifecycle | HIGH | Foundation for everything. 15 items, only 4 DONE. Must be first. |
| **Building State Machine** (construct, operate, destroy) | Players must build huts, temples, training buildings. No buildings = no game | HIGH | 34 items, 5 DONE. Construction, occupants, population growth, wood consumption all TODO |
| **Building Placement UI** (ghost preview, validation) | Players need to place buildings; this is the primary player action in early game | MEDIUM | Terrain flattening/footprints already DONE, need validation + UI ghost |
| **Population System** (hut spawning, housing capacity) | Braves spawn from huts = core economy loop. No population growth = no army | MEDIUM | Hut spawn rates, population cap, housing capacity per level |
| **Wood Economy** (gathering, carrying, storage, costs) | Wood is the only resource. Buildings consume wood, braves gather it from trees | MEDIUM | 6 items all TODO. Tree interaction, wood values, building costs |
| **Mana System** (generation, spell costs, cooldowns) | Mana = spellcasting currency. Generated by population. Core positive feedback loop | MEDIUM | Per-unit-type generation rates, per-activity rates, housing level bonuses |
| **Spell System** (at least 8-10 core spells) | Spells are the primary combat mechanic and the god-game identity. Without spells the game is a bland RTS | HIGH | 21 spells total, all TODO. Prioritize: Burn, Blast, Lightning, Convert Wild, Flatten, Land Bridge, Shield, Teleport |
| **Combat System** (melee damage, death, projectiles) | Units must fight and die. Melee is PARTIAL, needs projectiles for drum towers and spells | HIGH | Melee formula PARTIAL. Projectiles, knockback, building combat all TODO |
| **AI/Scripting System** (bytecode interpreter) | Campaign levels have scripted AI opponents. Without AI, levels are empty sandboxes | HIGH | 18 items all TODO. Most complex single system: 200+ opcodes, personality traits, decision making. This is the long pole. |
| **HUD/UI** (minimap, spell bar, info panels, mana display) | Cannot play without seeing mana, population, spell availability, or minimap | HIGH | 26 items, only 2 DONE (keyboard + mouse input). Need minimap, spell bar, mana bar, population display, unit info |
| **Person State Machine** (enter/exit building, train, gather wood, drown) | Units need to interact with buildings and resources. Currently only move/idle/attack | MEDIUM | 44+ states, handful DONE. Key gaps: EnterBuilding, ExitBuilding, Training, GatherWood, Housed, Drown |
| **Training System** (warrior, spy, preacher, super warrior) | Campaign requires specialized units. Training huts convert braves to specialists | MEDIUM | Conversion timers, wood/mana costs, building behavior flag 0x01 |
| **Victory/Defeat Conditions** | Must be able to win or lose a level | LOW | Check every 16 ticks: all enemies eliminated = win, player eliminated = lose |
| **Campaign Progression** (25 levels, unlocks) | Must advance between levels with correct unlock progression | LOW | Level table, completion flags, spell/building discovery unlocks |
| **Discovery System** (stone head worship) | Spells and buildings are unlocked by worshipping stone heads on campaign levels | MEDIUM | 6 items all TODO. Worship progress, spell/building grants |
| **Health Bars** | Cannot assess unit/building health without visual feedback | LOW | 4 bar types (standard, mana, building, training) |
| **Terrain Modification** (flatten, raise/lower for spells) | Multiple spells and building placement modify terrain. Flatten Land used constantly | MEDIUM | Height modification function TODO. Water level changes for Erosion/Swamp/Land Bridge |
| **Save/Load System** | Campaign players expect to save progress. Losing 30+ minutes of gameplay = rage quit | MEDIUM | Full game state serialization. 860KB state per save. |
| **Font/Text Rendering** | Status messages, menu text, unit counts all require text | MEDIUM | Font loading (3 sizes), character rendering, string table (0x526 strings) |
| **Menu System** (main menu, campaign select, load game) | Must be able to start the game, select levels, load saves | MEDIUM | Minimum: main menu, single player, campaign, load game, options |

### Differentiators (Beyond Original Game)

These would set Pop3 apart from just running the original in DOSBox/compatibility mode. Not required for playable campaign but valuable for the project's purpose.

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| **Modern resolution support** | Original locked to 640x480/800x600/1024x768. Modern widescreen is a huge QoL win | MEDIUM | Already partially handled by wgpu rendering. Need UI scaling |
| **Uncapped frame rate** | Original was locked to game tick rate for rendering. Smooth 60/120fps rendering | LOW | Separate render from sim tick. Partially exists already |
| **Cross-platform** (macOS, Linux, Windows) | Original was Windows-only. macOS/Linux players have no option | LOW | Already achieved via wgpu + winit. This is a current strength |
| **Improved pathfinding visualization** | Debug overlay for paths, walkability. Helps development and is interesting for fans | LOW | Walkability overlay already DONE. Path arrows TODO |
| **Replay system** | Record game commands for replay. Valuable for debugging determinism and community content | MEDIUM | Deterministic RNG already DONE. Need command recording/playback |
| **Quality-of-life controls** | Modern RTS conveniences: control groups, right-click orders, attack-move | MEDIUM | Original had limited hotkeys. Careful not to break faithfulness |

### Anti-Features (Deliberately NOT Building Yet)

| Feature | Why Requested | Why Problematic | Alternative |
|---------|---------------|-----------------|-------------|
| **Multiplayer/Networking** | Original had multiplayer; fans want it | Requires lockstep sync, netcode, lobby, desync detection. Massive complexity (10 items all TODO). Blocks nothing for single-player | Complete single-player campaign first. Multiplayer is post-v1. Deterministic RNG already works, which is the hardest prerequisite. |
| **All 93 visual effect types** | Full visual fidelity | Huge art/rendering effort for non-gameplay impact. Many effects are cosmetic (particles, weather) | Implement gameplay-critical effects only (spell impacts, death, construction). Cosmetic effects later. |
| **Creature System** (bears, wolves, eagles, etc.) | Present in original game | Only appears on a few levels. No campaign progression depends on creatures. 3 items all TODO | Defer entirely. Creatures are flavor, not mechanics. Add after core campaign works. |
| **Vehicle System** (boats, airships) | Used in later campaign levels | Complex: 9 states, boarding/exit, production buildings, flying physics. Only needed for ~5 late-game levels | Implement after core campaign levels 1-15 work. Vehicles needed for levels ~16+ |
| **Full scenery interaction** (tree burning, portals, bridges) | Present in original | Most scenery is decorative. Tree harvesting is table stakes (wood economy), but burning/falling animations are cosmetic | Implement wood gathering from trees (table stakes). Defer visual tree destruction. |
| **Tutorial system** | Original had tutorial levels | Custom tutorial code, special scripting. Players who know the original do not need it | Skip entirely for v1. Target audience knows the game. |
| **Custom key bindings** | QoL feature | Complex input system (207 actions, binary format). Not needed for playability | Use hardcoded sensible defaults. Add rebinding post-v1. |
| **Multi-language support** | Original supported 12 languages | Multi-byte character handling, string table system. English-only is fine for initial release | English only. String table infrastructure enables later localization. |
| **Debug/Cheat system** | Useful for development | Not needed for players. Dev can use code-level cheats | Use Rust debug builds with hardcoded dev tools instead of reimplementing the cheat system |

```
                       |
                       +--requires--> [Building State Machine] (AI builds)
                       +--requires--> [Spell System] (AI casts)
                       +--requires--> [Combat System] (AI fights)
                       +--requires--> [Person State Machine] (AI commands units)

[HUD/UI] --independent--> (can be built in parallel)
    |
    +--requires--> [Font/Text Rendering]
    +--requires--> [Mana System] (display values)

[Victory/Defeat] --requires--> [Combat System] (elimination check)

                       +--requires--> [Discovery System] (unlock tracking)

[Save/Load] --requires--> [Core Object System] (serialize all state)

[Vehicle System] --requires--> [Core Object System]
                 +--requires--> [Person State Machine] (boarding states)
                 +--requires--> [Building State Machine] (production)
```

### Dependency Notes

- **Core Object System is the foundation:** Every gameplay system depends on a unified object pool with proper lifecycle, cell linkage, and spatial queries. This must be implemented first, before any gameplay feature.
- **Building System unlocks the economy loop:** Population growth, wood economy, training, and mana generation all flow through buildings. Buildings are the second priority after the core object system.
- **AI/Scripting is the long pole:** It depends on nearly everything else (buildings, spells, combat, person states) and is the most complex single system (200+ bytecode opcodes). It must be started early but will be completed last.
- **HUD/UI is independently parallelizable:** It reads game state but does not affect it. Can be developed alongside gameplay systems.
- **Vehicle System has a hard dependency wall:** Requires object system, person states, and building production all working. Only needed for late campaign levels. Natural deferral point.

## MVP Definition

### Launch With (v1 -- Playable Campaign)

Minimum for a player to complete the 25-level campaign:

- [ ] **Core Object System** -- unified pool, cell grid, spatial queries. Everything else blocks on this.
- [ ] **Building System** -- construction, operation, destruction, occupants, population growth, wood consumption
- [ ] **Building Placement** -- ghost preview, terrain validation, player click-to-place
- [ ] **Person State Extensions** -- enter/exit building, gather wood, training, housed, drown
- [ ] **Wood Economy** -- tree harvesting, wood carrying, building storage, consumption
- [ ] **Population System** -- hut spawning, housing capacity, spawn rate bands, population cap
- [ ] **Mana System** -- per-unit generation, spell costs, cooldowns
- [ ] **Spell System (core 12)** -- Burn, Blast, Lightning, Convert Wild, Flatten, Land Bridge, Shield, Teleport, Swamp, Erosion, Earthquake, Volcano
- [ ] **Combat System** -- melee damage formula, projectiles, knockback, building combat, death
- [ ] **AI/Scripting** -- bytecode interpreter, script loading, 200+ opcodes, personality traits
- [ ] **HUD** -- minimap, spell bar, mana/population display, unit info panel, health bars
- [ ] **Menu System** -- main menu, campaign select, load game
- [ ] **Victory/Defeat** -- elimination checks, reincarnation timer
- [ ] **Campaign Progression** -- 25-level sequence, completion flags
- [ ] **Discovery System** -- stone head worship, spell/building unlocks
- [ ] **Save/Load** -- full state serialization, quicksave
- [ ] **Terrain Modification** -- height changes for spells and building placement
- [ ] **Font/Text Rendering** -- 3 font sizes, string table, status messages
- [ ] **Minimal Effect System** -- spell impact visuals, death effects, construction effects (subset of 93 types)

### Add After Validation (v1.x)

Features to add once core campaign is playable:

- [ ] **Remaining 9 spells** -- Whirlwind, Insect Plague, Invisibility, Hypnotism, Firestorm, Ghost Army, Angel of Death, Blood Lust, Armageddon
- [ ] **Vehicle System** -- boats and airships for late campaign levels
- [ ] **Creature System** -- bears, wolves, eagles for flavor on specific levels
- [ ] **Full Effect System** -- environmental, particle, weather effects
- [ ] **Audio System** -- 3D positional audio, SFX, SoundFont music
- [ ] **Scenery Interaction** -- tree burning, portal activation, bridge mechanics
- [ ] **Spy System** -- disguise, detection, sabotage
- [ ] **Preacher Conversion** -- converting enemy units

### Future Consideration (v2+)

Features to defer until single-player is solid:

- [ ] **Multiplayer** -- lockstep networking, lobby, desync detection
- [ ] **Replay System** -- command recording/playback
- [ ] **Modding Support** -- not in original scope
- [ ] **Custom Key Bindings** -- 207-action binding system
- [ ] **Multi-language** -- 12-language string table support
- [ ] **Tutorial Levels** -- custom scripted tutorial

## Comparable Project Analysis

| Aspect | OpenRA (C&C Reimplementation) | OpenMW (Morrowind) | Pop3 Approach |
|--------|-------------------------------|---------------------|---------------|
| Core loop first | Yes: harvest, build, fight before campaign | Yes: movement, combat, quests before full world | Same: object system -> economy -> combat -> AI -> campaign |
| AI approach | Modular bot system, built after core gameplay | NPC AI via scripting engine | Bytecode interpreter matching original scripts. Must be faithful. |
| Campaign pacing | Skirmish mode first, campaign later | Main quest playable early, side content later | Campaign IS the product. No skirmish mode needed. |
| Audio timing | Added after gameplay was solid | Added incrementally | Defer audio to P2. Silent gameplay is acceptable for testing. |
| Save/Load | Added relatively late | Core requirement from early on | Must have for campaign. Losing 30min progress is unacceptable. |
| Multiplayer | Core feature from start (different from Pop3) | Single-player only (like Pop3 v1) | Explicitly deferred to post-v1. |
